# AGENTS.md (2026/05/25)

## 基本方針
- 指示されない限りソースコードは変更しないこと。
- 既存の設計・コーディングスタイルを尊重すること。
- 変更は必要最小限にすること。
- 無関係なリファクタリングを行わないこと。
- コメントや空白の変更のみを大量に行わないこと。

- C言語の場合は C99 を使う
- 新しい compiler warning を発生させないこと。
- インデントは 4 spaces
- 新規依存ライブラリを追加する前に許可をとること。
- ファイル名変更を行う前に許可をとること。
- ディレクトリ構成変更を行う前に許可をとること。
- 日本語文字コードは UTF-8 を使うこと。

## コミット
- commit -m のメッセージは必ず英語にすること。
- コミットメッセージは簡潔に書くこと。
- 1コミット1目的を原則とすること。

## Pull Request
- PRのタイトルは必ず英語とすること。
- PRの本文は必ず日本語で作成すること。
- PR本文には以下を書くこと：
  - 変更内容
  - 変更理由
  - 動作確認方法

## テスト
- 変更した箇所に関係するテストを優先して実行すること。
- テストが失敗している場合は原因を説明すること。
- 実行できなかったテストがある場合は理由を説明すること。

## 禁止事項
- 秘密情報を追加しないこと。
- ライセンス表記を削除しないこと。
- 大規模な自動整形を行わないこと。

- 数値精度を不必要に変更しないこと。
- 浮動小数点演算順序を不必要に変更しないこと。
- 並列化(OpenMP等)の有無を勝手に変更しないこと。
- 倍精度(double)を単精度(float)へ変更しないこと。
- 数値アルゴリズムを変更する場合は理由を説明すること。
- SIMD化や最適化により数値結果が変わる可能性がある場合は明記すること。

## Git ブランチ
- ブランチ名は `codex/` から始まり、続いて特徴を表す英単語を４個以下でハイフンでつないで名付けること。
- ブランチ名に含める英単語は、略記を使ってよい。
- ブランチ名はなるべく短い名前にすること。
- ブランチ名には空白を含めないこと。
- ブランチ名は英語とすること。
